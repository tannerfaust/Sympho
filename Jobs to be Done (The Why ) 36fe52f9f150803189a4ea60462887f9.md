# Jobs to be Done (The "Why")

**The "Capture" Job:** When I stumble upon a critical knowledge gap mid-project, I want to instantly log that topic without breaking my flow state, so I can return to it when I actually have dedicated study time.

**The "Deconstruction" Job:** When I decide to tackle a massive, intimidating subject, I want to see the underlying logical sequence of prerequisites, so I don't feel overwhelmed or stupid by starting in the wrong place.

**The "Triage" Job:** When my backlog of "things to learn" hits 50+ items, I want to map them against my important goals at the moment, so I spend my limited weekend hours studying what actually moves the needle.

**The "Resource Convergence" Job**: When I finally sit down for a dedicated, two-hour study block, I want all my scattered PDFs, specific documentation links, and video tutorials attached directly to the exact topic node I'm working on, so I spend 100% of my time actually absorbing the material instead of hunting through chaotic bookmarks and browser tabs.

**The "Context Switching" Job:** When I return to a project after a few days away, I want the app to provide a high-level summary of where I left off and what my next immediate learning objective is, so I can eliminate the "ramp-up" time and regain flow instantly.

**The "Knowledge Debt" Job:** When I realize I’ve been ignoring a foundational skill because it’s "boring" but it’s now causing repeated bugs in my work, I want the system to surface that specific skill as a critical-path dependency, so I can pay off my "knowledge debt" before it halts my project.

**(the weakest) The "Retention" Job:** When I finish a tough module, I want a system that prompts me to review the core concepts weeks later, so the time I spent learning isn't wasted by forgetting.
